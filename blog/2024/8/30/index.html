避免​

随着年龄的增长，两种我需要避免的变化：

  • 固执自己对世界的看法，忽视其他年龄段人群的世界观与需求。

对于未知领域的抗拒也许是为了避免自己掉入知识黑洞。人毕竟不可能把生活中的每个事情都研究透彻。但对于通用工具热情的缺失是思维停滞的危险信号。

  • 夸夸其谈自己的想法，实际上不明就里。

开发​

项目具有高度重复性，可以创建一个通用的第三方库。项目开发中，代码简洁易读是第一位。

通勤中学习​

指定计划是碎片化时间首要目标。最好是看一本书，看一段TED，自己的收藏。这样的学习有明确的结果，能让你在学习中获得成就感。

避免泛信息输入：看新闻/刷推荐。泛信息有效信息减少，且容易停不下来。好处：提供一些情绪价值（但是可以专门看脱口秀等来获得情绪价值）

适当的半精准信息输入：读书/刷博客/看github等

通勤完成后记得及时整理记录。

随着职业地位的提升，一些不必要的内耗被替代。高中时非常在意一本还是二本，如今在大厂面前都统一成了双非。想要进入世界一流的企业，需要有对应的世界排名。一些校级、省级的奖项已经变成兴趣爱好的证明，国家级的奖项方能说明具有一定才能。只有世界级的排名与项目才能说明你是对的人。

不能证明自己的人，会被划分出去，聪明是一种阶层。

人生按顺序存在不同阶段，每个阶段有不同的评价标准。学生时期主要的评价体系是院校。工作初期评价体系变成了薪资待遇，人到中年评价体系变成事业与家庭。

作为三非程序员（非985非211非计算机专业），目前我希望我的下个阶段：

精通Python：django flask 消息队列 算法 熟悉Js/Ts：Vue、React 熟悉Fushion360、Blender建模软件，积极了解新技术如：Rust、Ai模型方向等 Github开源项目XX作者（20k） 长期在Discord、Github使用英语/中文作为工作语言，与世界各地优秀的愿意友好沟通的开发者合作。

努力推动世界人民团结，吸收古今中外各家所长以解决当下与未来的问题。解决问题时候实事求是，能解决问题的理论就是好理论。

夹缝中游​

最近，我间断性的学习了建模技术，我发现了在最专业的软件（往往学习成本更高）与最简单的软件（往往效果不佳），我倾向与中间款，能快速给我一个及格的作品

教育供应商​

原本用于考核学生的被用来考核指导老师，而指导老师便会更倾向于选择提供答案的厂商。

对于机构与老师而言，他们的核心目标共通点是消课，即提供适合长度的PPT/教案，让老师可以照本宣科，如果做成文字版本，那么可能老师并不一定能讲的很好，而如果提供视频版本，那么内容一旦有勘误和补全就会变得难于处理。

在AI时代，可以使用AI智能课的形式来辅助学生学习，即:将电脑屏幕二分，一侧显示课件，另一侧提供AI助手，每个人都可以根据自己对课件的不理解的部分向AI助手提问。这样，机构和教师的工作量就变成了考核学生，处理学生的各种问题。

一些名师的水平毋庸置疑，但我国有多少教师和机构的水平在AI之下？

迷你工作室（1-5人）​

我最近在学习一些新的技术，我发现了一个非常适合我的工作环境，就是迷你工作室。迷你工作室是一个小型的工作室，通常由1-5个人组成，他们一起完成一个项目，这个项目可以是一个软件，也可以是一个游戏，也可以是一个网站。

成立工作室是因为开发票的需要，能开发票才能接到更多的单。

在ai时代，我们有非常多的机会。那么第1个方向就是加入大公司和他们一起研究通用人工智能。这个环节是卷生卷死的，强度非常大，需要耗费大量的资源。

那么第2个方向适用于中小型公司，可以选一些垂类人工智能也就是在一些很细分的领域。做出你自己的模型，譬如说单片机的新产品开发那么通用人工智能在这方面可能只会用到市面上一些优质的资料，因为他为了完美，必须要做一些取舍，但是你做垂类模型的话，你可以把你能找到所有的资料全部喂给他。

那么除了卷人工智能模型本身以外呢，做人工智能模型的拓展和应用，比如说给他识图，让它与我们的自然生活产生交互。目前为止主流的小爱音箱，小米音箱，都没有提供比较直接的控制方式，那么小米汽车上是可以通过语音去控制家庭设备和车窗的开关的，那么小米在未来是非常有潜力的。

那这样的拓展和应用其实是是一个技术爆发的结果，这有赖于我们的高速通信，我们无处不在的信号塔。还有我们丰富的硬件，我们充足的电力。

经济下行不好创业或找工作的原因：

投资不易回报，企业扩张减慢，换言之，之前繁荣社会上有一些岗位是不创造价值的，探索一些可能性。而在经济下行时，这些岗位便被取消掉。

科技停滞，没有好的增长点。各行各业技术成熟处于内卷状态。要么足够资深参与内卷，要么足够全面节省人员开支。

岗位在消失：如电动车时代的汽修工，不好找工作是因为汽修岗位已经逐步消失了。传统的网页开发也在模板化中消失了。数字人、爬虫、安全、AI嵌入等新出现的岗位是增长点，趁早转型。

业务的一定要确保需求真实且长期的存在，有很多老板认为业务需求很大，实际上根本没有足够的客户与客观的项目金额。

物品的稀缺程度决定价格，但是并不能证明稀缺物品更能让人感到快乐。

譬如：马拉松结束后，一瓶2元的水，和10元的面包。

快乐源自需求是否被满足，因此，找到客户，并在对的场景下遇到你的客户。满足他的需求，过的额外的收入。

真相也是稀缺的，帮助人们筛选、过滤、总结信息是当下非常有价值的事情，信息不等于真相，真相是代价高昂的稀有信息，而且有时错综复杂，令人不适。

确认需求存在之后，思考壁垒：

品牌壁垒​

奢侈品的品牌壁垒是精神属性，记作： 精神属性 ——> 奢侈品 决策复杂 ——> 装修、政策代办 犯错成本 ——> 药品、整容、奶粉 技术壁垒 ——> 专利矩阵

实现的很慢很难（但客观来说有很多假的技术壁垒，譬如各种技术的实现.真的技术壁垒:博士、数据的规模效应、研发投入资金） 许可壁垒 ——> 政府关系 转换壁垒 ——> 社交软件 关系、数据迁移、用户习惯、特别权益、深度绑定

对于TO B业务来说，转化成本和先发优势很重要。

  • 网络效应:用户用多，用户体验越好

全局网络效应：外卖的网络效应是被圈在单元模型城市里的，这种优先击穿不要做基本盘 局部网络效应：网络商城，office，社交软件

  • 规模效应:用户越多，成本越低

定制是反规模效应的，提升规模效应可以拆业务流程和标准化。把关键环节标准化。

  • 合规需要：消防许可、营业执照、计算机网络营业执照
  • 办公场地：饮水机、打印机、公司内网、桌椅、插排、水电费、办公场地、冰箱、微波炉。
  • 招聘：公司需要购买社保提供一些必要的便利，一个员工的实际支持约工资的1.5~2.5倍。

如果我们把感情看作0和游戏。那么总会会有人因长期利益受损而对这段关系感到不满。 因此，只有当一方的所求不是另一方的付出时，0和才会被打破。即：某一行为是二者都收益的。 企业与员工关系亦是如此，企业提供平台员工发挥，员工将在平台创造的利润与企业分成。

平淡生活的解药​

平淡生活的源自外部世界没有新的输入。当我们一份工作做了很多年没有上涨，没有变化。内部的平淡便开始显现，只有当我们不断的成长：工作的升值，可以支撑前往更远的地方，更好的家庭。认识更多的朋友。

将Cursor添加到右键菜单​

通过脚本读取当前用户名，并在桌面生成名为AddCursorToContextMenu.reg的注册表文件。如果提示权限不足，可以在桌面找到该文件，以管理员身份运行即可。

import os
import subprocess

# 获取当前用户名
username = os.getenv('USERNAME')

# 获取当前用户的桌面路径
desktop_path = os.path.join(os.path.expanduser("~"), 'Desktop')

# 构造 Cursor.exe 的路径
cursor_path = fr"C:\Users\{username}\AppData\Local\Programs\Cursor\Cursor.exe"

# 生成注册表内容
reg_content = f"""Windows Registry Editor Version 5.00

[HKEY_CLASSES_ROOT\\*\\shell\\Open with Cursor]
@="Open with Cursor"
"Icon"="\"{cursor_path}\""
[HKEY_CLASSES_ROOT\\*\\shell\\Open with Cursor\\command]
@="\"{cursor_path}\" \"%1\""

[HKEY_CLASSES_ROOT\\Directory\\shell\\Open with Cursor]
@="Open with Cursor"
"Icon"="\"{cursor_path}\""
[HKEY_CLASSES_ROOT\\Directory\\shell\\Open with Cursor\\command]
@="\"{cursor_path}\" \"%1\""

[HKEY_CLASSES_ROOT\\Directory\\Background\\shell\\Open with Cursor]
@="Open with Cursor当前文件夹"
"Icon"="\"{cursor_path}\""
[HKEY_CLASSES_ROOT\\Directory\\Background\\shell\\Open with Cursor\\command]
@="\"{cursor_path}\" \"%V\""

[HKEY_CURRENT_USER\\Software\\Classes\\CLSID\\{{86ca1aa0-34aa-4e8b-a509-50c905bae2a2}}\\InprocServer32]
@=""
"""

# 生成 .reg 文件的完整路径，保存在桌面
reg_file_path = os.path.join(desktop_path, "AddCursorToContextMenu.reg")

# 将注册表内容写入 .reg 文件，使用 UTF-8 编码
with open(reg_file_path, "w", encoding="utf-8") as file:
    file.write(reg_content)

print(f".reg 文件已保存到: {reg_file_path}")

# 执行 .reg 文件来导入注册表
try:
    subprocess.run(['regedit.exe', '/s', reg_file_path], check=True)
    print("注册表文件已成功导入。")
except subprocess.CalledProcessError as e:
    print(f"导入注册表文件时出错: {e}")
